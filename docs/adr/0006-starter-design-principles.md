# A starter's example slides are a demonstration surface, not filler

A starter ships a small example deck (`index.html` + its `theme.css` and `notes.js`), and we treat those slides as a thing that has to *earn its keep* — the developer's first impression of the product and the pattern the AI imitates afterwards — rather than as placeholder content to be discarded. Three consequences fall out of that. The example slides use **deliberately distinct layout archetypes** (a cover, a dense grid, a sparse statement, a close), never N variations of centred text: variety is not decoration, it *is* the pitch — slidash's core value is that the AI adapts the layout to each slide's content, and a deck of look-alike slides would silently contradict it. The slides only ever rely on **real slidash conventions** (`.slide`, `data-steps`/`data-step`, `data-thumb-step`); a starter never demonstrates a bespoke mechanic of its own, because whatever the example does, the agent will reproduce. If a starter seems to "need" a markup mechanic the conventions don't offer, that is a conventions decision to be made on its own merits — it does not enter through a starter's example slides. And the example deck doubles as **human onboarding**: it carries the guidance meant for the *person* — what the keys do, that everything is theirs to edit, what the product is for — which `AGENTS.md` deliberately omits because that file speaks only to the AI. The slides are where the human learns; `AGENTS.md` is where the agent does.

We rejected the obvious alternative — ship a neutral, reusable template and let the content be throwaway lorem. It is cheaper to write but it teaches the agent nothing (lorem has no layout to imitate), wastes the strongest first impression the product gets, and tempts starters toward a single safe layout that undersells the one thing slidash does that a slide builder cannot. The trade-off we accept is that the example content is bespoke per starter and gets overwritten as soon as the developer starts their real talk — but while it exists it demonstrates, and that is the point. A minor naming consequence in the same spirit: starter tokens are named by **role, not by hue** (`--accent`, not `--indigo`), so an agent that re-colours the theme doesn't leave a token lying about what it holds. (Related: the files a starter contributes and the vendored, `file://`-openable model — ADR-0001; speaker notes as a separate example file — ADR-0002; how a starter tints the conveniences rather than restyling them — ADR-0004.)
